# 👗 Fashion Size Tracker Bot

Telegram-бот, который отслеживает наличие нужного размера на сайтах модных брендов и присылает уведомление, когда размер появится в наличии.

**Поддерживаемые сайты:** Zara, Massimo Dutti, Loewe, и любые другие (через универсальный парсер).

---

## 🚀 Деплой на Railway (бесплатно, ноутбук не нужен)

### Шаг 1 — Создай Telegram-бота

1. Открой Telegram, найди [@BotFather](https://t.me/BotFather)
2. Отправь `/newbot`
3. Придумай имя и username для бота
4. Скопируй токен (выглядит как `1234567890:ABCdef...`)

### Шаг 2 — Загрузи код на GitHub

1. Зарегистрируйся на [github.com](https://github.com) (если нет аккаунта)
2. Нажми `+` → `New repository`, назови его `fashion-tracker`
3. Загрузи все файлы из этой папки в репозиторий

### Шаг 3 — Задеплой на Railway

1. Зарегистрируйся на [railway.app](https://railway.app) (вход через GitHub)
2. Нажми `New Project` → `Deploy from GitHub repo`
3. Выбери репозиторий `fashion-tracker`
4. После создания проекта перейди в `Variables` и добавь:
   ```
   BOT_TOKEN = твой_токен_от_BotFather
   ```
5. Railway автоматически запустит бота!

### Шаг 4 — Проверь

Открой своего бота в Telegram и отправь `/start`.

## 💬 Как пользоваться

| Действие | Что делать |
|---|---|
| Начать отслеживать | Отправь ссылку на товар |
| Выбрать размер | Нажми на кнопку с размером или напиши вручную |
| Посмотреть список | `/list` |
| Остановить отслеживание | Нажми 🛑 в уведомлении или в списке `/list` |

## 📁 Структура файлов

```
fashion-tracker/
├── main.py          — Telegram бот (хендлеры, шедулер)
├── scrapers.py      — Парсеры сайтов (Zara, Massimo, универсальный)
├── database.py      — SQLite база данных
├── requirements.txt — Python зависимости
├── Procfile         — Конфиг для Railway/Heroku
└── railway.json     — Конфиг для Railway
```

---

## ⚠️ Важные нюансы

- **Zara/Massimo** — используют официальный Inditex API, работает стабильно
- **Loewe и другие люкс бренды** — используется HTML-парсер. Иногда сайты меняют структуру или блокируют парсеры — в этом случае нужно будет обновить `scrapers.py`
- База данных (`trackings.db`) хранится на сервере Railway. При пересоздании проекта отслеживания сбросятся
